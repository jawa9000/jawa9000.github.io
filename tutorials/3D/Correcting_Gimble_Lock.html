Correcting Gimble Lock Using Set Driven Keys

Written by Brian Immel

Set up the Joints

Create skeleton setup similar to image to the right.

Select the Shoulder_upper and group it. Name the group Shoulder_group. Note: if you create this group and it not centered on the Shoulder_upper joint, center it so that it is! To avoid gimble lock, we need another pivot to draw rotational information so that when we rotate one axis on top of another, it will not lock.

Create the Controller

Create the letter S using a NURBS curve (Create > Text > option box).

In the outliner, select the curve1 and unparent it. Delete Text_S_1. The reason we are deleting the groups is to avoid any issues about selecting orders and assigning new attributes to the wrong node later on.

Select curve1 and rename it to Shoulder_Control.

Add Attributes

Using the Add Attribute window add the following attributes to the Shoulder_Control:

  • Shoulder UD Min: -10, Max 10, Default 0. UD stands for Up and Down.
  • Shoulder TW Min: -10, Max 10, Default 0. TW stands for Twist.
  • Shoulder FB Min: -10, Max 10, Default 0. FB stands for Forward and Backward.
SDK Setup

Your Attribute Editor should look like Figure 2 when you select Shoulder_Control.

Next we are going to set up the Set Driven Keys using the SDK window. Using the Set Driven Key window, load Shoulder_Control as the Driver.

Select the Shoulder_group and Shoulder_upper joint and load them as the Driven.

Set Driven Keys: ShoulderUD

Highlight ShoulderUD in the Driver section and highlight rotateX of Shoulder_upper in the Driven section and hit Key. RotateX should control the up and down motions of the shoulder rotation. If this rotation does not perform this function, select the appropriate rotation and set the SDKs as necessary.

Set ShoulderUD to 10.

Rotate Shoulder_upper rotateX to 90.

Hit Key in the Set Driven Key window.

Set ShoulderUD to -10.

Rotate Shoulder_upper rotateX to -90.

Hit Key in the Set Driven Key window.

Set ShoulderUD to 0.

Set Driven Keys: ShoulderTW

Highlight ShoulderTW in the Driver section and highlight rotateZ of Shoulder_upper in the Driven section and hit Key. RotateZ should control the twisting motion of the entire arm from the shoulder down. If this rotation does not perform this function, select the appropriate rotation and set the SDKs as necessary.

Set ShoulderTW to 10.

Rotate Shoulder_upper rotateZ to 90.

Hit Key in the Set Driven Key window.

Set ShoulderTW to -10.

Rotate Shoulder_upper rotateZ to -90.

Hit Key in the Set Driven Key window.

Set ShoulderTW to 0.

Set Driven Keys: ShoulderFB

Highlight ShoulderFB in the Driver section and highlight rotateY of Shoulder_group in the Driven section and hit Key. RotateY of the Shoulder_group should rotate the entire hierarchy forward and backward. If this rotation does not perform this function, select the appropriate rotation and set the SDKs as necessary.

Set ShoulderFB to 10.

Rotate Shoulder_upper rotateY to 90.

Hit Key in the Set Driven Key window.

Set ShoulderFB to -10.

Rotate Shoulder_upper rotateY to -90.

Hit Key in the Set Driven Key window.

Set ShoulderFB to 0.

Another Option

Another option to help bend the shoulder up/down and forward/backward is to set up the Shoulder_lower in the SDKs as well. Give the Shoulder_upper joint the majority of the rotational values and the Shoulder_lower joint a fraction of the total rotations. Example: if the total rotation of the shoulder moving forward is 90, the give the Shoulder_upper joint 60 and Shoulder_lower joint 30.
