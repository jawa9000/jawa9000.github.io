Overview

This document shows entry-level programmers and data scientists how to set up a SQL query using the [table] table to generate a list of wiki pages that have been abandoned (no assigned owner). This tutorial also shows you how to sort the results by ordering the results in either an ascending or descending manner. This query is part of a Learning SQL series designed to help you get started with SQL and create your own dashboard filled with queries to report on your wiki's health.

This tutorial assumes you are familiar with using the [Query UI] to write and test the query used in this document and are using the [wiki_database] database set in your [Query UI]. See our documentation for how to use [Query UI] to learn the basics of this interface.

Abandoned page query

Copy and paste this code snippet into your [Query UI] and click the Submit button.

SELECT
page_title, page_owners
FROM [table]
WHERE
ds='<DATEID>'
AND page_title LIKE '[wiki_root]%'
AND page_owners is NULL
LIMIT 500
ORDER BY page_title ASC

This query looks through all the pages in a given wiki root and all the pages in it's hierarchy ([wiki_root]%) as of today's date (ds='<DATEID>') and looks for any page where the page property for page owner is empty (NULL). Additionally, the query limits the number of results to 500 (LIMIT 500). If you get 500 results and suspect there may be more, you can increase this number but this will cause the query to run slower and take up more computing resources.

Ordering results

The query order results in ascending order by the name of the wiki pages (ORDER BY page_title ASC). If you want to reverse the order of the results, change ORDER BY page_title ASC (ascending order) to ORDER BY page_title DES (descending order).

Note: if this query returns empty, then that means all your wiki pages under the provided wiki root have page owners.
